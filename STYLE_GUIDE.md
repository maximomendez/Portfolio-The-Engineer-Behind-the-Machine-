# Style Guide — The Engineer Behind the Machine

Style guide to keep Maximo Mendez’s current portfolio consistent.

---

# 1. Concept

The portfolio should feel like a premium interface for operating a high-performance digital machine.

## Core Idea

> The Engineer Behind the Machine

## Main Message

> Full Stack Developer specialized in Backend, Automation, and AI. I build the systems, workflows, and intelligence that make digital products perform better.

The experience should communicate:

- Real engineering.
- Systems thinking.
- End-to-end technical capability.
- Backend, automation, data, and AI integration.
- Performance and operational precision.

The website should not feel like a traditional portfolio template or a generic visual landing page.

---

# 2. Visual Personality

The aesthetic combines references from:

- High-performance automotive cockpit.
- Technical dashboard.
- Futuristic HUD.
- Operational interfaces.
- System signals and telemetry.

## Visual Principles

- Dark-first.
- High contrast and extreme readability.
- Precise and technical composition.
- Color used with functional intention.
- Subtle glow to indicate activity or energy.
- Modular and connected interfaces.
- Real-time operating system feel.

## Avoid

- Excessive neon.
- Decorative gradients.
- Futuristic elements without purpose.
- Long or cinematic animations.
- Generic copy such as:
  - “Passionate developer”
  - “Beautiful websites”
  - “Creative solutions”

## Rules

- System labels in uppercase.
- Wide tracking in labels and HUD.
- Long text prioritizes readability.
- Maintain a clear hierarchy and few typographic weights per component.

---

# 7. Components and Patterns

## Ignition

Initial activation screen.

### Patterns

- Dark background.
- Subtle technical grid.
- Brackets and system signals.
- Soft scan lines.
- Industrial-style primary CTA.

### Primary CTA

- Red border.
- Transparent background.
- Uppercase mono.
- Hover with color inversion.

---

## Dashboard / Cockpit

Main cockpit-style control panel experience.

### Principles

- Immersive responsive stage.
- Modular HUD.
- Compact and precise information.
- Status indicators.
- Performance metrics.
- Navigation integrated into the cockpit.

### Rules

- Keep overlays lightweight.
- Prioritize readability.
- Avoid visual saturation.

---

## Engine

Section representing backend architecture and system flow.

### Patterns

- Central engine as operational core.
- Connected modules.
- Visible request flow.
- Active signals and nodes.

### Color Semantics

| Color | Meaning |
|---|---|
| Red | Core / Logic |
| Blue | APIs / AI / Data |
| Green | Automation / Operational |

---

## Panels / HUD Cards

### Base

- Dark or glass background.
- Soft borders.
- Moderate blur over images.
- Accent lines only when the module is active.

---

## Buttons

### Primary

- Red border.
- Uppercase mono.
- Solid red hover with dark text.

### Secondary

- Translucent background.
- Subtle border.
- Hover with red accent.

### Ghost

- No background.
- Muted text.
- Light hover state.

---

## Badges and Labels

- Uppercase.
- Wide tracking.
- Compact.
- Functional use, not decorative.

---

# 8. Content and Tone

The content should sell technical capability and systems thinking.

## Tone

- Professional.
- Clear.
- Technical.
- Confident.
- Direct.

---

## Correct Messages

- “I build backend systems that automate business workflows.”
- “I connect APIs, data, and AI into production-ready tools.”
- “I design systems that reduce manual work and improve operational speed.”

---

## Avoid

- “Passionate developer.”
- “Amazing digital experiences.”
- “Creative and motivated.”
- “Modern beautiful websites.”

---

## Every Section Should Answer

- What I build.
- What problem it solves.
- Why it matters.
- What impact it creates.
- What action the user should take.

---

# 9. Motion

## Principles

- Motion communicates activation and state.
- Interactions should feel fast and precise.
- Infinite animations are only used for signals and telemetry.
- Avoid cinematic or distracting motion.
- Respect `prefers-reduced-motion`.

---

## Timing

| Use | Duration |
|---|---:|
| Instant | `100ms` |
| Hover | `200ms` |
| Standard | `400ms` |
| Section Reveal | `600ms` |

---

# 10. Accessibility and Performance

## Accessibility

- Semantic HTML.
- High contrast.
- Keyboard navigation.
- States that do not depend only on color.
- Accessible tooltips.
- Descriptive alt text for images.

---

## Performance

- Prioritize Server Components.
- Client Components only when there is real interaction.
- Optimize images.
- Lazy-load heavy visuals.
- Avoid unnecessary dependencies.
- Keep cockpit and engine fluid on mobile and desktop.

---

# 11. Implementation

## Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- Framer Motion
- Lucide React
- Vercel

---

## Best Practices

- Separate editable content in `src/data`.
- Centralize tokens and animations.
- Reuse existing utilities.
- Avoid unnecessary abstractions.
- Keep a mobile-first approach.

---

# 12. Final Objective

The portfolio should feel like:

- an operational digital machine,
- a modern engineering interface,
- and a real demonstration of technical capability.

It should not impress only through aesthetics.

It should communicate:

- judgment,
- architecture,
- automation,
- performance,
- and the ability to build real end-to-end systems.
